Build 1064:

    - Archipelago: Save as PNG now gives you the option to save
      at a zoom level other than 100%.  For including in documents
      and PowerPoint presentations and such, smaller zoom levels
      generally look better.
      Special thanks to Haig F. Krikorian [haig.f.krikorian@boeing.com]
    - BNA: ThingLogic lifecycle updated.  Now, BNAComponent will call
      setComponent() with a non-null parameter when adding a logic
      and setComponent() with a null parameter when removing one.
      ThingLogicAdapter responds to this by calling init() and
      destroy() methods on the logic, respectively.  destroy() is
      no longer a logic interface method although it remains in
      ThingLogicAdapter.

      name of the c2.fw interface on a brick that corresponds

      BrickID.InterfaceID hack.  The hack is still supported
      for now, but is deprecated.

      components to detect when a document is being closed or saved
      and gives them an opportunity to get work done before
      approving the change.
    - Archipelago: Archipelago now uses the NotifyDocs mechanism
      to store hints only when a document is about to be saved
      or closed, or when the Archipelago window is closed
      instead of when the window loses focus.
      Special thanks to Haig F. Krikorian [haig.f.krikorian@boeing.com]
    - Archipelago: You can now save as PNG in a type view.
      Special thanks to Haig F. Krikorian [haig.f.krikorian@boeing.com]
    - File Manager/Archipelago: File Manager and Save as PNG now store

Build 1055:
  Eric M. Dashofy [edashofy@ics.uci.edu]
    - Got rid of most deprecation warnings in the source.
    - Archipelago: Fixed a latent bug in LinkMappingLogic that
      could cause an infinite loop when moving a link endpoint from
      one interface to another; symptom was the link endpoint
      jumping back and forth between the two interfaces.
    - Added the ability to rename (e.g. change the URI of) an
      open xArch document.  Renaming sends out an XArchFileEvent
      indicating the rename. This means that components should
      not rely on the URI of an open document being totally stable;
      they should either update their code to rely on root-element
      ObjRefs (which are stable), handle the rename file event
      properly, or both.
    - Updated critics, ArchEdit, and other tools to use root-
      element ObjRefs rather than URIs to refer to open
      documents.
    - File Manager: File manager now detects changes to open
      architecture descriptions and highlights them with an
      asterisk as in most modern multi-document applications.
      It also asks the user if they want to save unsaved changes
      on close document or exit.
    - Deprecated a few functions that had "URL" in their names
      xArchADT and replaced them with "URI".  Updated calling
      components to use the non-deprecated functions.
    - Fixed a latent race condition in the critics framework
      that could cause wrong issues to be displayed if an issue
      was removed before a critic artist was able to render it.

    - Added a new mechanism for dealing with Java-style event
      listeners in EBIWrapperUtils; replaced existing state-
      change message processors in various ArchStudio components
      with this mechanism.  Due to the fact that this mechanism
      does not throw/catch Exceptions when presented with irrelevant
      events (like many ArchStudio event processors did), there
      has been a significant (~18%) speedup to ArchStudio operations
      that involve sending/receiving a lot of events.

Build 1048:
  Eric M. Dashofy [edashofy@ics.uci.edu]
    - Archipelago: Peers are now optionally responsible for determining
      the local bounding box of their target Things for drawing
      optimization.
      Special thanks to Justin Erenkrantz [justin@erenkrantz.com]
    - Archipelago: Update to BNAComponent's painting mechanism to use
      peer bounding boxes along with the clip rectangle to
      optimize painting to only those regions that have changed.
      Also changed the way painting is held from a setHoldPainting()
      method in the component to a pair of (optional) methods in the
      model: beginBulkChange() and endBulkChange().  This means that
      bulk change begin/end notifications become part of the
      event stream from the model and therefore don't get into weird
      race conditions with the direct calls on the component.
      Special thanks to Justin Erenkrantz [justin@erenkrantz.com]
    - Archipelago: Edit interface-to-signature mapping widget now
      displays correct title and also displays current interface-to-
      signature mapping when started.
    - Archipelago: Glass mode/gradient mode plugins are now
      somewhat better-behaved.
    - xArchADT now caches context objects so you don't burn a whole
      bunch of ObjRefs when you create lots of them.
    - xArchADT finally emits events in response to transactions the
      right way: if changes took place inside a transaction, they
      are emitted as both regular XArchFlatEvents and ALSO grouped
      up inside an XArchTransactionEvent.  If a change took place
      outside a transaction, it is emitted as a single XArchFlatEvent
      and ALSO in a 1-item group in an XArchTransactionEvent.  This
      means that components can either listen for XArchFlatEvents
      or XArchTransactionEvents, and still get all the events only
      once.
    - Slightly improved the XadlUtils class to add a couple more
      methods regarding XLinks, also refactored TypeWrangler and some
      of Archipelago to use these.
    - Refactored the BrickSelectorDialog and TypeSelectorDialog into
      the more generic StructureSelectorDialog and TypeSelectorDialog
      in the xadlutils package.  These provide nice, tree-based
      dialogs to select a structural element (brick or interface)
      or a type.  NewSIMContextMenuPlugin now uses the
      StructureSelectorDialog interface rather than rolling its own
      and is much more compact now.
    - TypeWrangler is now included in the archstudio.xml file.
      My bad!
      Special thanks to Kari Nies [kari@ics.uci.edu]
    - Updated AEM and JavaImplementationCritic to support change in
      cardinality of "implementation" element in xADL 2.0.
    - Configurator now puts spaces around the whole classpath instead
      of just individual elements on Windows; apparently this works
      properly and solves a problem some users were having caused by
      spaces in their environment classpath.

Build 1019:
  Eric M. Dashofy [edashofy@ics.uci.edu]
    - Fixed a bug in IdTable.java that caused RuntimeExceptions on
      some valid set(..., "href", ...) calls or
      clear(..., "href", ...) calls.
    - Various and sundry performance enhancements across the board.
      Put in a serious performance fix for OneThreadPerBrick message
      handler; now the handler only wakes up threads that are actually
      affected by a message being sent rather than every thread in
      the architecture.  Approximate speedup for all these
      enhancements is between 40 and 70%.
    - For performance reasons, split up XArchADTC2Component and
      XArchTransactionsC2Component into alternatives, rather than
      XArchTransactionsC2Component being a "layer in front of"
      XArchADTC2Component.  Now, architects using an architecture
      ADT should choose *either* XArchTransactionsC2Component (for
      applications needing transactions) *or* XArchADT (for
      applications not requiring transactions).  Theoretically, this
      reduces flexibility slightly, as you can no longer add
      transaction support on the fly, but this functionality is
      rarely, if ever, needed.  Speedup is another 40% or
      thereabouts.
    - I got tired of writing boilerplate subclasses of
      NamedPropertyMessage every time I wanted a new message type,
      so I wrote a simple generator.  Try it.  Run:
      java c2.util.MessageCodeGenerator
    - AEM has been completely rearchitected.  AEM Driver now supports
      all the new functionality.  AEM now responds to changes in
      bound architecture descriptions (i.e. it monitors transaction
      events from XArchTrans) and attempts to co-evolve the
      architecture along with the description.
    - Several fixes in the c2.fw framework regarding what happens
      when you remove a brick.  Previously, brick removal was
      broken in several places, mostly because parts of the framework
      received a brickRemoved event *after* the brick had been
      removed, at which point there was not enough information left
      to clean up.  Split that event into two events, brickRemoving
      (before removal) and brickRemoved (after removal), and fixed
      appropriate parts of the framework to listen for the correct
      event.  Similar fix for weldRemoved->weldRemoving/weldRemoved.
    - Added an efficient isAttached(...) function to
      XArchFlatInterface.
    - Fixed the LinkCritic.
